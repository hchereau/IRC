1) 클라이언트 객체는 소켓 스트림이란 객체를 지닌다
소켓 스트림 - 클라이언트의 데이터 입출력 스트림 관리, 읽기 쓰기 각각 2개 버퍼

IRC 서버가 존나게 돌아가는 동안
fd_set 에 등록된 fd 값에 변화가 생기면 블로킹 해제

2) 읽기 쓰기에 대한 오해
읽기 recv : 커널->유저 공간으로 데이터 복사, 네트워크에서 들어온 바이트 받기
읽기는 그냥 데이터(바이트) 읽기 이며 
이걸 요청으로 만드는 건 파서 역할이다

쓰기 send : 유저 공간 -> 커널-> 네트워크로 전송, 응답 바이트 보내기
1. poll : 읽을 수 있음 
2. recv/read 바이트를 버퍼에 가져옴
3. 버퍼에 쌓음
4. 파서가 해석

소켓에서 읽는 건 그냥 바이트 덩어리 (스트림!) ***
